2. Conversation State Machine

How the agent routes and handles multi-turn conversations.

stateDiagram-v2
    [*] --> Router: Customer message
    Router --> Reserve: intent = reserve
    Router --> FAQ: intent = faq
    Router --> Cancel: intent = cancel
    Router --> Decline: intent = off_topic

    state Reserve {
        [*] --> CollectName
        CollectName --> CollectDate
        CollectDate --> CollectTime
        CollectTime --> CollectPartySize
        CollectPartySize --> CollectRequests
        CollectRequests --> CheckAvailability
        CheckAvailability --> Confirm: available
        CheckAvailability --> Suggest: unavailable
    }

    state Cancel {
        [*] --> LookupPhone
        LookupPhone --> ShowMatches
        ShowMatches --> ConfirmCancel
    }

    Reserve --> [*]: Booking confirmed
    FAQ --> [*]: Answer returned
    Cancel --> [*]: Cancelled
    Decline --> [*]: Polite refusal

6. RAG Pipeline

How customer questions get answered using restaurant knowledge files.

7. Security Layers

Multi-layered defense pipeline — every message passes through before reaching the agent.

flowchart LR
    A["User Input"] --> B{"Length under 2000?"}
    B -->|no| X["Blocked"]
    B -->|yes| C{"Injection Pattern?"}
    C -->|match| X
    C -->|clean| D{"Rate Limit OK?"}
    D -->|exceeded| X
    D -->|ok| E["Agent Processes"]
    E --> F{"Output Filter"}
    F -->|"API key or path leaked"| X
    F -->|clean| G["Response to User"]
